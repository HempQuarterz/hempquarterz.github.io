All4Yah Full Component Audit — 16 Components Analyzed

18 issues found across core + discovery components. Select the areas you want to fix first.

CRITICAL — Users Can't Access EOTC Texts (5 issues)

1

ManuscriptViewer — Hardcoded 12 manuscripts

Lines 86-99: Missing CHARLES, WIKISOURCE, BUDGE, GEEZ. Language mapping has no Ge'ez case. Users literally cannot select Ethiopian texts.

2

BookSelector — Missing EOTC testament groups

Only shows OT, NT, Deuterocanon. Books with "Ethiopian Canon" / "Pseudepigrapha" testaments are invisible. Can't navigate to ENO, JUB, ASI, 1MQ, 2MQ, 3MQ, KNG.

3

CanonicalFilterPanel + Badge — Wrong tier counts

Tier 4 says "1 Ethiopian Orthodox text (Kebra Nagast)" but we now have 7 EOTC books across 4 manuscripts. Misleading to users.

4

NetworkGraph + Timeline — Missing EOTC in bookOrder

Both components hardcode 66-book bookOrder mapping. EOTC books cause undefined errors in BFS traversal and era classification. Timeline shows wrong "chronological" order.

5

Cross-References API — Hardcoded OT/NT book lists

crossReferences.js lines 300-316 has hardcoded book arrays. EOTC book codes not recognized as valid targets.

HIGH — Dark Theme Violations + Data Gaps (5 issues)

6

ParallelPassageViewer — White backgrounds + hardcoded WEB

Uses background: white, color: #333 throughout. Also hardcodes 'WEB' manuscript — can't view EOTC parallels. Two issues in one component.

7

CrossReferenceBadge — White tooltip on dark background

Tooltip uses backgroundColor: '#fff', color: '#333'. Unreadable against dark glass ConsolidatedPanel.

8

ThematicDiscoveryPanel — Only 70 verses in search pool

Hardcodes 7 sample books (GEN, PSA, ISA, MAT, JHN, ROM, REV), loads 10 verses each. Semantic search misses 99.97% of the corpus including all EOTC texts.

9

Language CSS — No Ge'ez font mapping

getLanguageClass() maps geez → english-text fallback. Ge'ez text renders with Cardo (English serif) instead of Noto Serif Ethiopic. Font is loaded but never applied.

10

Restoration API — No Ge'ez language entry

restoration.js has no geez case. Divine name restoration skips EOTC texts. (May be OK since Ge'ez preserves original names.)

MEDIUM — Performance + Polish (4 issues)

11

ThematicDiscovery — Client-side ML embeddings

Generates embeddings on-the-fly with Transformers.js (~100MB model). Blocks UI, no caching. Should pre-compute or use simpler keyword matching.

12

Timeline — Sequential verse loading

Loads 8 verses one-by-one instead of parallel Promise.all(). Also uses fake "chronological" timestamps.

13

ManuscriptCarousel — English-only title detection

Book title verse filtering assumes English patterns. Ge'ez titles would either not filter or strip actual verse 1.

14

InterlinearViewer — No EOTC alignment data

Defaults to WLC→WEB. Shows "no alignments" for EOTC. Won't crash but offers nothing for Ethiopian texts.
